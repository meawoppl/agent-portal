# iOS push registration bridge (plan §15, E4)

`PushRegistrationBridge.swift` converts an APNs device token into a portal
push subscription (`POST /api/push/subscriptions`, `platform: "apns"`). It is
the iOS counterpart of the web `PushManager` subscribe flow and the Android
FCM bridge.

> **Status: scaffold — not compile-verified.** CI has no macOS builder and no
> Apple credentials yet (plan items F2/F3). Nothing references this file until
> it is added to the generated Xcode project, so it cannot break existing
> builds. Treat the wiring below as the integration checklist for whoever
> lands F2.

## Why it lives here (not in `gen/apple`)

`mobile/src-tauri/gen/` is regenerated by `cargo tauri ios init` and is
gitignored. Source that must survive regeneration lives in `mobile/ios/` and
gets added to the Xcode project as part of integration.

## Wiring checklist (after `npm run ios:init`)

1. **Add the file** to the generated project: open
   `mobile/src-tauri/gen/apple/*.xcodeproj`, add `mobile/ios/
   PushRegistrationBridge.swift` to the app target (reference, don't copy).
2. **Entitlement**: add the *Push Notifications* capability to the app target
   (`aps-environment` = `development` for debug, `production` for release).
3. **AppDelegate forwarding** — three calls:

   ```swift
   // After the shell's auth handoff has produced a mobile JWT:
   PushRegistrationBridge.shared.start(config: .init(
       backendBaseURL: URL(string: shellBackendURL)!,
       bearerToken: shellAuthToken   // the shell store's AUTH_TOKEN_KEY value
   ))

   func application(_ application: UIApplication,
                    didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
       PushRegistrationBridge.shared.didRegister(deviceToken: deviceToken)
   }

   func application(_ application: UIApplication,
                    didFailToRegisterForRemoteNotificationsWithError error: Error) {
       PushRegistrationBridge.shared.didFailToRegister(error: error)
   }
   ```

   The shell owns the JWT (its store's `auth_token` key — see
   `mobile/src-tauri/src/lib.rs`); pass it by value. Call `start` on every
   launch: APNs rotates tokens, and re-POSTing is idempotent server-side.
4. **Server side**: the backend delivers to `apns` subscriptions only when
   its APNs transport is configured (p8 key etc. — C7, already merged);
   otherwise delivery intent is logged by the fallback transport, which is
   enough to verify the registration path end-to-end-minus-Apple.

## Contract

Request body mirrors `shared::api::RegisterPushSubscriptionRequest`:

```json
{
  "platform": "apns",
  "endpoint_or_token": "<64-char lowercase hex device token>",
  "device_label": "Matt's iPhone (iPhone)"
}
```

`p256dh`/`auth` are Web Push-only and omitted. Auth is the mobile JWT as
`Authorization: Bearer …` (accepted by the subscriptions API since the
bearer-auth work, #1294).
